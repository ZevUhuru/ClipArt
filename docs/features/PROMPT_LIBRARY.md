# Prompt Library

## Overview

A persistent prompt library that stores AI-generated and user-written animation prompts per source image. Replaces the previous ephemeral suggestion system where prompts were generated, used once, and lost.

## Problem

1. Users generate 5 AI prompt suggestions, use 1, and the other 4 vanish — wasting the value of that Gemini call.
2. Every "Suggest prompts" click hits Gemini even for the same image, adding unnecessary latency and cost.
3. If one user gets great suggestions for a community image, other users can't benefit from them.
4. Users have no way to save, collect, or reuse prompts they've found effective.

## Solution

Store every generated prompt in a `animation_prompts` table tied to the source image. The "cache" is a natural byproduct — if prompts exist for an image, return them instead of calling Gemini. Prompts accumulate over time into a growing library per image.

## Architecture

```
User clicks "Suggest prompts"
  → POST /api/animate/suggestions { generationId, imageUrl, regenerate?, duration? }
    → Check animation_prompts table for this generation_id
    → Cache HIT:  return stored prompts (sorted by use_count, then recency)
    → Cache MISS: call Gemini (duration-aware) → store 5 new prompts → return them
  → "Regenerate" sets regenerate: true → always calls Gemini, adds 5 NEW rows

User clicks "Animate" with a prompt
  → POST /api/animate { ..., promptId }
    → If promptId provided, increment use_count on that row
```

## Database Schema

```sql
CREATE TABLE public.animation_prompts (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  generation_id uuid NOT NULL REFERENCES public.generations(id) ON DELETE CASCADE,
  created_by uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  title text NOT NULL,
  prompt text NOT NULL,
  is_ai_generated boolean NOT NULL DEFAULT true,
  is_public boolean NOT NULL DEFAULT true,
  use_count integer NOT NULL DEFAULT 0,
  created_at timestamptz NOT NULL DEFAULT now()
);
```

Key columns:
- `generation_id` — ties prompts to the source image (the `generations` record)
- `created_by` — who generated/wrote this prompt
- `is_ai_generated` — distinguishes AI suggestions from user-written prompts
- `use_count` — tracks how many times a prompt was actually used to create an animation
- `is_public` — controls visibility to other users

## Key Decisions

- **Prompts accumulate, never delete** — "Regenerate" adds new prompts alongside existing ones. The library grows richer over time.
- **Cross-user benefit** — Prompts are `is_public: true` by default. User B sees prompts generated by User A for the same image.
- **Popular prompts surface first** — Results ordered by `use_count DESC` so battle-tested prompts appear at the top.
- **Session cache on client** — `useRef<Map>` keyed by `${generationId}:${duration}` avoids redundant API calls when switching between images or durations in one session.
- **`promptId` tracking** — When user selects a stored prompt and animates, the `promptId` is passed to the animate API to increment `use_count`.
- **Duration-aware suggestions** — The selected clip duration (5–15s) is sent to the suggestions API and injected into both the Gemini system prompt and user message. Shorter clips get tighter, single-beat prompts; longer clips get multi-beat choreography with richer scene direction. The system prompt includes pacing rules and word-count targets scaled to the duration.

## Duration-Aware Prompt Generation

AI suggestions adapt to the selected animation duration. The `duration` parameter flows through:

1. **Frontend** — `fetchSuggestions()` sends the current slider value in the API request body
2. **API** — `/api/animate/suggestions` validates duration (5–15, default 5) and passes it to Gemini
3. **System prompt** — `getAnimationSystemPrompt(duration)` injects pacing guidance:
   - **5s**: Single motion beat, 40–80 words, tight and punchy
   - **10s**: 2–3 motion beats, 60–120 words, camera transitions allowed
   - **15s**: Full mini-scene, 80–150 words, setup → buildup → climax → settle
4. **User prompt** — The Gemini user message explicitly states the clip duration and complexity expectations
5. **Client cache** — Keyed by `${generationId}:${duration}` so switching durations fetches fresh suggestions

**Note:** The DB cache (`animation_prompts`) does not currently store duration, so it returns the first-generated set regardless. The client-side cache handles duration-specific lookups. A `duration` column can be added to `animation_prompts` in the future if duration-specific DB caching becomes necessary.

## Caching Layers

1. **Client (session)** — `Map<generationId:duration, PromptSuggestion[]>` in a `useRef`. Survives image/duration switching within one session. Zero latency.
2. **Server (Supabase)** — Persisted `animation_prompts` rows. Survives deploys, shared across all users. ~50ms latency. Not currently duration-specific.
3. **AI (Gemini)** — Only called on cache miss or explicit "Regenerate". ~1-3s latency, ~$0.003/call. Duration-aware system + user prompts.

## Future Roadmap

### Phase 2: Prompt Favorites & Collections

- Users can "star" / bookmark prompts they want to save
- "My Saved Prompts" section in the user profile or My Creations page
- Prompt collections (e.g., "my best camera prompts", "character action prompts")

### Phase 3: Community Prompt Index

- Dedicated `/prompts` browse page showing popular prompts across all images
- Filter by style, mood, category (cinematic, playful, dramatic, etc.)
- Sort by popularity (`use_count`), recency, or rating
- Each prompt links to example animations it produced

### Phase 4: Prompt Ratings & Quality Signal

- Users upvote/downvote prompts after seeing the animation result
- Quality score derived from: use_count, upvotes, completion rate (did the animation actually succeed?)
- Surface "Editor's Pick" or "Top Rated" badges on high-quality prompts
- Use quality signal to improve the Gemini system prompt over time

### Phase 5: Cross-Image Templates

- Identify prompts that work well across many images (e.g., "character waves hello" works for any character image)
- Auto-suggest universal templates based on image category/style
- "Works with X images" indicator on universal templates
- Template marketplace where users can publish and share prompt templates

### Phase 6: Prompt Sharing & Social

- Shareable prompt links (e.g., `clip.art/prompts/abc123`)
- "Used by X people" social proof
- Prompt leaderboard: top prompt creators
- Fork/remix: take an existing prompt and modify it

### Phase 7: Image Generation Prompt Suggestions

- Extend the same system to the `/create` page for image generation
- `getGenerationSystemPrompt()` already scaffolded in `promptKnowledge.ts`
- New `/api/generate/suggestions` endpoint
- Separate `generation_prompts` table or expand `animation_prompts` with a `type` column
- Users describe what they want vaguely → AI refines it into 5 detailed prompts

### Phase 8: Prompt Analytics & Learning

- Track which prompts produce the best animations (user ratings + engagement)
- Feed winning patterns back into the Gemini system prompt
- A/B test different system prompt versions
- Personalized suggestions based on user's past prompt preferences
