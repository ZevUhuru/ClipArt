# Admin CMS

Internal admin panel for managing images, categories, and content at `clip.art/admin`.

## Access

Protected by `is_admin` flag on the `profiles` table. The admin layout (`app/admin/layout.tsx`) checks this server-side before rendering any admin page. Non-admin users are redirected to `/`.

To grant admin access:

```sql
UPDATE profiles SET is_admin = true WHERE email = 'you@example.com';
```

## Pages

### Dashboard (`/admin`)

Overview stats: total generations, today's count, revenue, purchase count, active categories. Server component, no caching (`revalidate = 0`).

### Images (`/admin/images`)

Paginated table of all generations. Supports:

- **Search** by title or prompt (case-insensitive `ILIKE`)
- **Filter** by category slug
- **Filter** by visibility (public/hidden)
- **Inline toggle** for public/hidden status
- **Delete** with confirmation (removes R2 object + DB row)
- **Edit link** to full detail page

### Image Edit (`/admin/images/[id]`)

Full metadata editor for a single generation:

- **Title** — clean, display-ready name (auto-generated by classifier, editable)
- **URL Slug** — used in `/{category}/{slug}` detail page URL. "Auto-generate from title" button
- **Description** — SEO description shown on detail page
- **Category** — dropdown of all DB categories. Changing category moves the R2 object to the new path and updates the `image_url`
- **Public toggle** — controls gallery visibility
- **Delete** — permanent removal from DB and R2

Shows original prompt for reference.

### Categories (`/admin/categories`)

CRUD for the category system:

- **Create** — name + slug, with "Auto-generate SEO fields with AI" toggle. When enabled, Gemini Flash generates h1, meta title, meta description, intro, SEO paragraphs, and suggested prompts
- **Edit** — all fields editable: slug, name, h1, meta title, meta description, intro
- **Active/Inactive toggle** — inactive categories don't appear in navigation or sitemap
- **Delete** — removes category row (images in the category are NOT deleted)

## API Routes

All admin API routes verify `is_admin` via Supabase auth before processing.

| Method | Route | Purpose |
|--------|-------|---------|
| `GET` | `/api/admin/images` | List generations (paginated, searchable, filterable) |
| `GET` | `/api/admin/images/[id]` | Get single generation |
| `PATCH` | `/api/admin/images/[id]` | Update title, slug, description, category, is_public |
| `DELETE` | `/api/admin/images/[id]` | Delete generation + R2 object |
| `GET` | `/api/admin/categories` | List all categories |
| `POST` | `/api/admin/categories` | Create new category (with optional AI SEO) |
| `PATCH` | `/api/admin/categories/[id]` | Update category fields |
| `DELETE` | `/api/admin/categories/[id]` | Delete category |

## R2 Object Movement

When an image's category is changed via PATCH, the API:

1. Copies the R2 object from `{old-category}/{filename}` to `{new-category}/{filename}`
2. Deletes the old R2 object
3. Updates `image_url` in the DB to the new path
4. Calls `revalidatePath` for both old and new category pages
