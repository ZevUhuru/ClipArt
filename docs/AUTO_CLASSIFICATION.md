# Auto-Classification

Every generated image is automatically classified using Gemini Flash (text model) to produce clean metadata before saving to the database.

## How It Works

After the image is generated by Gemini (image model), but before the DB insert, the generate route calls `classifyPrompt()` from `src/lib/classify.ts`.

### Input

- User's raw prompt (e.g. `"cute littel asian girl in chinese new year outfit"`)
- Selected style (e.g. `"cartoon"`)

### Output

| Field | Description | Example |
|-------|-------------|---------|
| `title` | Clean, capitalized, typo-corrected (max 60 chars) | `"Cute Little Asian Girl in Chinese New Year Outfit"` |
| `category` | Best-matching slug from DB categories table | `"free"` (or a more specific one if it exists) |
| `description` | SEO-friendly description (100-160 chars) | `"Adorable cartoon girl in traditional Chinese New Year dress..."` |
| `slug` | URL-friendly slug from the title | `"cute-little-asian-girl-chinese-new-year-outfit"` |

### Model

Uses `gemini-2.0-flash` (text-only) with `temperature: 0.1` for consistent, deterministic output. Cost is negligible (~$0.0001 per call).

### Category Matching

The classifier fetches the current list of active category slugs from the DB at call time. This means:

- New categories added via the admin CMS are immediately available for classification
- No code deploys needed to teach the classifier about new categories
- If no category fits, falls back to `"free"`

### Graceful Degradation

If the Gemini text call fails for any reason (rate limit, network, malformed response), the function returns a fallback using the raw prompt:

```
title: prompt.slice(0, 80)
category: "free"
description: prompt
slug: slugify(prompt)
```

Generation is never blocked by a classification failure.

## Category SEO Generation

`generateCategorySEO()` in the same file generates all SEO fields for a new category:

- h1, meta title, meta description
- Intro paragraph
- 2 SEO content paragraphs
- 3 suggested prompts

Used by the admin categories API when "Auto-generate SEO fields with AI" is enabled.

## Flow Diagram

```
User prompt + style
       │
       ▼
  Gemini Image → image buffer
       │
       ├──────────────────────┐
       ▼                      ▼
  classifyPrompt()      uploadToR2()
       │                      │
       ▼                      ▼
  { title, category,    image URL
    description, slug }       │
       │                      │
       └──────────┬───────────┘
                  ▼
         DB insert (generations)
         with clean metadata
                  │
                  ▼
         revalidatePath('/{category}')
```
